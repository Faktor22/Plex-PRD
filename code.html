Front-end / Back-end

Back-end

Tijdens het ontwikkelen van het PLeX platform hebben we meerdere vergaderingen gehad, waarbij we rekening hebben gehouden met het bestaande DeX platform. Mede daardoor is er een ontwerp tot stand gekomen, die in het onderstaande architectuur wordt weergegeven.

C1 Context

Er zijn twee type gebruikers:

  • Docenten(projectleiders) zijn docenten die projecten kunnen zien en projectlijsten kunnen managen.
  • Students zijn studenten die hun toegewezen projectlijst kunnen zien.

C2 Containers

Dit diagram geeft meer overzicht over de containers die PLeX heeft die samen de complete applicatie zijn.

  • Front-end betreft de user interface waar de gebruiker projectlijsten kan managen of inzien.
  • API Back-end verwerkt alle request van de front-end.
  • Database is verantwoordelijk voor het opslaan van de data. Dit wordt gedaan door de back-end.

C3 Components

Onderstaande diagram geeft meer inzicht van de API Back-end.

Request handlers:

  • Project controller handelt alle requests af voor het het managen van de projecten.
  • Projectlist controller handelt alle requests af voor het managen van de projectlijsten.

Logic:

  • Project service managed de projecten.
  • Projectlist service managed de projectlijsten.

Data:

  • Project repository bewaard en ontvangt project data.
  • Projectlist repository bewaard en ontvangt projectlist data.
